# 快速启动指南

## 一、环境准备

### 1.1 安装Docker
```bash
# macOS
brew install docker docker-compose

# 启动Docker Desktop
```

### 1.2 配置环境变量

当前以根目录 `.env.example` 为唯一配置基准。

```bash
# 复制配置文件
cp .env.example .env

# 按需检查并修改本地配置
vim .env
```

关键配置项：
- `DATABASE_URL` - PostgreSQL 连接串
- `REDIS_URL` - Redis 连接串
- `CLAUDE_API_KEY` / `CLAUDE_BASE_URL` - Claude 文本和视觉能力
- `ZHIPU_API_KEY` / `ZHIPU_BASE_URL` - 智谱 Embedding 能力
- `QDRANT_URL` - Qdrant 地址
- `OSS_REGION` / `OSS_ACCESS_KEY_ID` / `OSS_ACCESS_KEY_SECRET` / `OSS_BUCKET_NAME` / `OSS_ENDPOINT` - OSS 配置
- `SECRET_KEY` - 应用密钥
- `DEBUG` - 调试开关

---

## 二、启动服务

### 2.1 启动基础服务
```bash
# 启动数据库、Redis、Qdrant
docker-compose up -d

# 查看服务状态
docker-compose ps
```

### 2.2 访问服务
- PostgreSQL: `localhost:5432`
- Redis: `localhost:6379`
- Qdrant: `http://localhost:6333`

---

## 三、初始化数据

当前目录只提供基础依赖和文档，不包含后端应用、迁移脚本和导入脚本。

如果你已经拿到后端仓库，建议在后端仓库中按下面顺序执行：
1. 创建数据库表或执行 Alembic 迁移
2. 初始化 Qdrant 集合
3. 导入分类和商品样例
4. 导入知识库并重建向量

这里的基础设施参数统一使用本目录 `.env.example` 里的变量名。

---

## 四、启动后端API

以下命令应在后端仓库中执行，不是在当前文档目录执行：

```bash
# 安装依赖
pip install -r requirements.txt

# 启动FastAPI
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

访问API文档：`http://localhost:8000/docs`

---

## 五、测试API

### 5.1 测试图像识别
```bash
curl -X POST http://localhost:8000/api/v1/ai/recognize-image \
  -H "Authorization: Bearer your-token" \
  -F "file=@test.jpg"
```

### 5.2 测试AI问答
```bash
curl -X POST http://localhost:8000/api/v1/ai/rag-chat \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer your-token" \
  -d '{"question": "F40用什么机油滤芯？"}'
```

## 六、常用命令

```bash
# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down

# 重启服务
docker-compose restart

# 清理数据（慎用）
docker-compose down -v
```

## 七、故障排查

### 问题1：端口被占用
```bash
# 查看端口占用
lsof -i :5432
lsof -i :6379

# 修改docker-compose.yml中的端口映射
```

### 问题2：Qdrant连接失败
```bash
# 检查Qdrant状态
curl http://localhost:6333/health

# 重启Qdrant
docker-compose restart qdrant
```

### 问题3：数据库连接失败
```bash
# 检查数据库状态
docker-compose exec db psql -U user -d hardware_store

# 查看数据库日志
docker-compose logs db
```
