# 数据库设计文档

> **数据库**：PostgreSQL 15
> **字符集**：UTF-8
> **时区**：UTC
> **产品定位**：船用五金 AI 识别与查询工具

---

## 一、设计目标

数据库仅服务于以下核心能力：
- 商品资料管理
- 分类浏览与搜索
- 拍照识别记录
- AI 问答记录
- 收藏与历史
- 知识库检索与维护
- 用户反馈闭环

不包含：
- 订单
- 支付
- 库存出入库
- 角色权限

---

## 二、数据库表结构

### 2.1 用户表 `users`

```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    phone VARCHAR(20) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    nickname VARCHAR(50),
    avatar_url VARCHAR(255),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_users_phone ON users(phone);
```

### 2.2 分类表 `categories`

```sql
CREATE TABLE categories (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    parent_id INT REFERENCES categories(id),
    level INT DEFAULT 1,
    sort_order INT DEFAULT 0,
    icon_url VARCHAR(255),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_categories_parent_id ON categories(parent_id);
CREATE INDEX idx_categories_sort_order ON categories(sort_order);
```

### 2.3 商品表 `products`

```sql
CREATE TABLE products (
    id SERIAL PRIMARY KEY,
    category_id INT REFERENCES categories(id),
    name VARCHAR(200) NOT NULL,
    brand VARCHAR(100),
    model_no VARCHAR(100),
    description TEXT,
    price DECIMAL(10,2),
    images JSONB,
    specs JSONB,
    compatibility JSONB,
    usage_scenarios TEXT,
    safety_tips JSONB,
    keywords JSONB,
    status VARCHAR(20) DEFAULT 'active',
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_products_category_id ON products(category_id);
CREATE INDEX idx_products_status ON products(status);
CREATE INDEX idx_products_model_no ON products(model_no);
CREATE INDEX idx_products_name_fts ON products USING gin(to_tsvector('simple', name));
CREATE INDEX idx_products_keywords_gin ON products USING gin(keywords);
CREATE INDEX idx_products_compatibility_gin ON products USING gin(compatibility);
```

### 2.4 AI 识别记录表 `ai_recognitions`

```sql
CREATE TABLE ai_recognitions (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id),
    image_url VARCHAR(255) NOT NULL,
    source VARCHAR(20) DEFAULT 'camera',
    confidence DECIMAL(4,3),
    recognition_result JSONB NOT NULL,
    matched_products JSONB,
    needs_more_images BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_ai_recognitions_user_id ON ai_recognitions(user_id);
CREATE INDEX idx_ai_recognitions_created_at ON ai_recognitions(created_at);
```

### 2.5 识别反馈表 `ai_recognition_feedback`

```sql
CREATE TABLE ai_recognition_feedback (
    id SERIAL PRIMARY KEY,
    recognition_id INT REFERENCES ai_recognitions(id) ON DELETE CASCADE,
    user_id INT REFERENCES users(id),
    feedback_type VARCHAR(50) NOT NULL,
    comment TEXT,
    correct_product_id INT REFERENCES products(id),
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_recognition_feedback_recognition_id
ON ai_recognition_feedback(recognition_id);
```

### 2.6 AI 对话会话表 `ai_conversations`

```sql
CREATE TABLE ai_conversations (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id),
    session_id VARCHAR(100) UNIQUE NOT NULL,
    title VARCHAR(200),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_ai_conversations_user_id ON ai_conversations(user_id);
CREATE INDEX idx_ai_conversations_updated_at ON ai_conversations(updated_at);
```

### 2.7 AI 对话消息表 `ai_conversation_messages`

```sql
CREATE TABLE ai_conversation_messages (
    id SERIAL PRIMARY KEY,
    conversation_id INT REFERENCES ai_conversations(id) ON DELETE CASCADE,
    role VARCHAR(20) NOT NULL,
    content TEXT NOT NULL,
    citations JSONB,
    recommended_products JSONB,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_ai_messages_conversation_id
ON ai_conversation_messages(conversation_id);
```

### 2.8 AI 回答反馈表 `ai_chat_feedback`

```sql
CREATE TABLE ai_chat_feedback (
    id SERIAL PRIMARY KEY,
    conversation_message_id INT REFERENCES ai_conversation_messages(id) ON DELETE CASCADE,
    user_id INT REFERENCES users(id),
    rating SMALLINT NOT NULL,
    comment TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_ai_chat_feedback_message_id
ON ai_chat_feedback(conversation_message_id);
```

### 2.9 收藏表 `user_favorites`

```sql
CREATE TABLE user_favorites (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id) ON DELETE CASCADE,
    product_id INT REFERENCES products(id) ON DELETE CASCADE,
    created_at TIMESTAMP DEFAULT NOW(),
    UNIQUE (user_id, product_id)
);

CREATE INDEX idx_user_favorites_user_id ON user_favorites(user_id);
```

### 2.10 搜索历史表 `search_histories`

```sql
CREATE TABLE search_histories (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id) ON DELETE CASCADE,
    query_text VARCHAR(200) NOT NULL,
    search_type VARCHAR(20) DEFAULT 'auto',
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_search_histories_user_id ON search_histories(user_id);
CREATE INDEX idx_search_histories_created_at ON search_histories(created_at);
```

### 2.11 商品知识库表 `product_knowledge`

```sql
CREATE TABLE product_knowledge (
    id SERIAL PRIMARY KEY,
    product_ids JSONB,
    content_type VARCHAR(50) NOT NULL,
    title VARCHAR(200) NOT NULL,
    content TEXT NOT NULL,
    engine_models JSONB,
    source_type VARCHAR(50),
    source_ref VARCHAR(255),
    version INT DEFAULT 1,
    status VARCHAR(20) DEFAULT 'active',
    content_hash VARCHAR(64),
    vector_id VARCHAR(100),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_product_knowledge_type ON product_knowledge(content_type);
CREATE INDEX idx_product_knowledge_status ON product_knowledge(status);
CREATE INDEX idx_product_knowledge_created_at ON product_knowledge(created_at);
CREATE INDEX idx_product_knowledge_product_ids_gin ON product_knowledge USING gin(product_ids);
CREATE INDEX idx_product_knowledge_engine_models_gin ON product_knowledge USING gin(engine_models);
```

### 2.12 知识库维护任务表 `knowledge_jobs`

```sql
CREATE TABLE knowledge_jobs (
    id SERIAL PRIMARY KEY,
    job_type VARCHAR(50) NOT NULL,
    source_file VARCHAR(255),
    status VARCHAR(20) DEFAULT 'pending',
    total_count INT DEFAULT 0,
    success_count INT DEFAULT 0,
    failed_count INT DEFAULT 0,
    error_summary TEXT,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_knowledge_jobs_status ON knowledge_jobs(status);
```

## 六、建模说明

- `products.price` 允许为空，因为工具型产品不强制成交价。
- `product_knowledge.product_ids` 使用 `JSONB`，用于一条知识关联多个商品。
- 识别和问答反馈单独建表，方便后续做错误样本回流。
- 对话拆成会话表和消息表，便于分页与反馈定位。
