# 船用五金 AI 识别与查询工具环境与部署文档

> **用途**：统一整理环境变量、账号资料、Docker 部署和运维要求
> **原则**：测试环境优先，生产环境后置

---

## 一、需要准备的资料

### 1.1 必须提供
- 后端仓库
- Flutter 仓库
- PostgreSQL 连接串
- Redis 连接串
- OSS 配置
- Qdrant 配置
- Claude API Key
- 智谱 API Key

### 1.2 建议提供
- 商品样例数据
- 分类初始数据
- 真实拍照样图
- 知识库 Excel / CSV / PDF

---

## 二、交付原则

- 优先给测试环境，不要先给生产环境
- 密钥不要写进 Git
- `.env` 必须加入 `.gitignore`
- 移动端不要放数据库密码、OSS 管理密钥、向量库密钥
- 变量命名和默认结构统一以 `.env.example` 为准

---

## 三、环境变量

### 3.1 基础配置

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `SECRET_KEY` | 应用密钥 | `replace_me` |
| `DEBUG` | 调试模式 | `False` |

### 3.2 数据库

当前默认使用 PostgreSQL。

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `DATABASE_URL` | 完整连接串 | `postgresql://user:password@localhost:5432/hardware_store` |

### 3.3 Redis

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `REDIS_URL` | 完整连接串 | `redis://localhost:6379` |

### 3.4 对象存储

当前 `.env.example` 使用阿里云 OSS：
- `OSS_REGION`
- `OSS_ACCESS_KEY_ID`
- `OSS_ACCESS_KEY_SECRET`
- `OSS_BUCKET_NAME`
- `OSS_ENDPOINT`
- `OSS_PATH_PREFIX`
- `OSS_USE_SSL`
- `OSS_USE_SIGN_URL`
- `OSS_SIGN_URL_EXPIRES`

### 3.5 向量库

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `QDRANT_URL` | 地址 | `http://127.0.0.1:6333` |

### 3.6 AI 配置

当前 `.env.example` 采用双供应商拆分：
- Claude 负责文本和视觉能力
- 智谱负责 Embedding

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `CLAUDE_API_KEY` | Claude API Key | `sk-ant-...` |
| `CLAUDE_BASE_URL` | Claude 服务地址 | `https://code.newcli.com/claude/ultra` |
| `ZHIPU_API_KEY` | 智谱 API Key | `replace_me` |
| `ZHIPU_BASE_URL` | 智谱服务地址 | `https://open.bigmodel.cn/api/paas/v4` |

说明：
- 如果后续要切成单供应商，再新增统一的 AI 抽象层，不要在文档里混用两套字段。
- 当前文档只描述 `.env.example` 已存在的变量，不再额外扩展 OpenAI 兼容字段。

### 3.7 其他

当前 `.env.example` 没有独立搜索引擎、CDN 或 API 基础地址配置。
如果后续增加 Elasticsearch、MinIO、Nginx 或 CDN，建议新开字段并同步更新 `.env.example`，不要只改 Markdown。

## 六、本地开发最小环境

最少需要：
- PostgreSQL
- Redis
- OSS
- Qdrant
- 一个可用的 Claude API Key
- 一个可用的智谱 API Key

建议联调顺序：
1. 跑通数据库
2. 跑通 Redis 和 Qdrant
3. 跑通图片上传
4. 跑通 Claude 识图 / 问答
5. 跑通智谱 Embedding 和知识检索

---

## 七、Docker 部署结构

```yaml
version: "3.8"

services:
  db:
    image: postgres:15

  redis:
    image: redis:7-alpine

  qdrant:
    image: qdrant/qdrant:latest
```

说明：
- 当前仓库里的 `docker-compose.yml` 只包含 PostgreSQL、Redis、Qdrant
- API 服务、Nginx、迁移脚本和索引脚本应放在后端仓库维护
- 如果后续把 API 也收进 Compose，先更新 `.env.example` 再更新部署文档

## 八、部署步骤

### 8.1 首次部署

```bash
git clone <your-repo>
cd <repo>
cp .env.example .env
docker-compose up -d
docker-compose ps
```

如果已经有后端仓库，再在后端仓库中继续执行：
- 安装依赖
- 执行数据库迁移
- 初始化知识向量
- 启动 FastAPI

### 8.2 更新部署

```bash
git pull
docker-compose up -d
```

## 九、监控与排障

常用命令：

```bash
docker-compose ps
docker-compose logs -f db
docker-compose logs -f redis
docker-compose logs -f qdrant
docker stats
df -h
free -h
```

常见问题：
- 数据库连接失败：先检查 `DATABASE_URL`
- 向量检索异常：先检查 `QDRANT_URL`
- 图片上传失败：先检查 OSS 配置和 bucket 权限

## 十、备份与安全

### 10.1 备份
- PostgreSQL 每日备份
- Qdrant 定期备份
- 重要知识库源文件单独存档

### 10.2 安全
- 所有密钥不要进 Git
- `.env` 加入 `.gitignore`
- OSS bucket 最好限制目录和权限
- 生产数据库最好限制 IP 白名单
- 文档字段必须和 `.env.example` 同步，避免出现多套配置口径
